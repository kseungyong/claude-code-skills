# Plan to Doc Skill

Claude Code에서 `/plan` 실행 후 계획 내용을 구조화된 설계 문서로 저장합니다.

---

## 언제 실행되는가

아래 중 하나라도 해당하면 즉시 이 스킬을 따릅니다:

- `/plan` 명령 실행 후 계획이 완성된 시점
- 사용자가 "계획 저장", "설계서 저장", "문서로 남겨줘" 등을 요청할 때
- 새 프로젝트 시작 시 Claude가 구현 계획을 수립한 직후

---

## 실행 절차

### Step 1. 프로젝트명 확인

현재 작업 디렉토리(프로젝트 루트)의 폴더명을 프로젝트명으로 사용합니다.

```bash
basename $(pwd)
```

프로젝트명에 공백이 있으면 하이픈(-)으로 대체합니다.

### Step 2. docs/ 폴더 생성

```bash
mkdir -p docs
```

### Step 3. 파일명 생성

형식: `{프로젝트명}-{YYYYMMDD}.md`

```bash
echo "$(basename $(pwd))-$(date +%Y%m%d).md"
```

예시: `kdsys-homepage-migration-20250307.md`

### Step 4. 설계 문서 작성 및 저장

아래 **문서 템플릿**을 기반으로 plan 내용을 채워 `docs/` 폴더에 저장합니다.

---

## 문서 템플릿

```markdown
# {프로젝트명} 구축 설계서

> 작성일: {YYYY-MM-DD}  
> 작성자: Claude (Claude Code /plan)  
> 버전: 1.0

---

## 1. 프로젝트 개요 및 목표

### 1.1 배경
{프로젝트가 필요한 배경과 현재 상황}

### 1.2 목표
{이 프로젝트로 달성하고자 하는 목표}

### 1.3 범위
{포함되는 작업 범위 / 포함되지 않는 범위}

---

## 2. 작업 단계별 상세 계획

| 단계 | 작업 내용 | 예상 소요 시간 |
|------|-----------|---------------|
| 1단계 | {작업명} | {시간} |
| 2단계 | {작업명} | {시간} |
| 3단계 | {작업명} | {시간} |

### 단계별 상세 설명

#### 1단계: {단계명}
- **목적**: {이 단계의 목적}
- **작업 내용**:
  - {세부 작업 1}
  - {세부 작업 2}
- **완료 기준**: {완료 조건}

#### 2단계: {단계명}
(동일한 형식으로 반복)

---

## 3. 예상 파일 / 폴더 구조

```
{프로젝트명}/
├── {폴더명}/
│   ├── {파일명}       # {역할 설명}
│   └── {파일명}       # {역할 설명}
├── {파일명}            # {역할 설명}
└── docs/
    └── {프로젝트명}-{YYYYMMDD}.md  # 이 설계서

## 저장 완료 후 출력 메시지

문서 저장이 완료되면 다음 형식으로 사용자에게 알립니다:

```
✅ 설계 문서가 저장되었습니다.
📄 경로: docs/{프로젝트명}-{YYYYMMDD}.md
```

---

## 주의사항

- 같은 날짜에 plan을 여러 번 실행하면 파일이 **덮어쓰기** 됩니다.
  - 두 번째 저장 시 버전 suffix를 붙입니다: `프로젝트명-20250307-v2.md`
- `docs/` 폴더가 `.gitignore`에 포함되어 있으면 Git에 커밋되지 않으니 확인이 필요합니다.
- 한글 프로젝트명은 파일명에 그대로 사용 가능하나, 영문 사용을 권장합니다.
